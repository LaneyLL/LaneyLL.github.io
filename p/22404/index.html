如何使用张量核（tensorcore）优化卷积(conv)

How to optimize convolution using TensorCores

作者:冯思远<https://github.com/Hzfengsy>
在本教程中，我们将演示如何在TVM中使用TensorCores编写一个高性能的卷积调度。在这个例子中，我们假设卷积的输入有一个大的批量。我们强烈建议先学习opt-conv-gpu教程。

TensorCore介绍

每个张量核提供了一个4x4x4矩阵处理阵列，其操作如下 D = A * B + C，其中A、B、C和D是4x4矩阵，如图所示。矩阵乘法输入A和B是FP16矩阵，而l累加矩阵C和D可以是FP16或FP32矩阵。
然而，CUDA程序员只能使用warp级原语wmma::mma_sync(acc_frag, a_frag, b_frag, acc_frag)在张量核上执行16x16x16的半精度矩阵乘法。在调用矩阵乘法之前，程序员必须显式地使用原语wmma::load_matrix_sync将数据从内存加载到寄存器中。NVCC编译器将原语转换为多个内存加载指令。在运行时，每个线程从矩阵A加载16个元素，从矩阵B加载16个元素。

Preparation and Algorithm 准备和算法

我们使用固定尺寸的256通道和14 x 14维的输入张量。批量大小为256。卷积滤波器包含512个大小为3 x 3的滤波器。我们使用步幅大小1和填充大小1来进行卷积。在本例中，我们使用NHWCnc内存布局。下面的代码定义了TVM中的卷积算法。

Memory Scope 内存范围

在传统的GPU调度中，我们有全局、共享和本地内存范围。为了支持TensorCores，我们添加了另外三个特殊的内存作用域:wmma.matrix_a,wmma.matrix_b,wmma.accumulator。在硬件上，所有的片段作用域存储在片上寄存器级别，与本地内存在同一个位置。

Define Tensor Intrinsic 定义张量内在

事实上，TensorCore是一种特殊的硬件操作。因此，我们可以使用 tensorize 来代替 TensorCore 指令的计算单元。首先我们需要定义张量的内禀。
在 TensorCore 中有四个基本操作: 填充片段fill_fragment、加载矩阵load_matrix、 mma 同步mma_sync和存储矩阵store_matrix。因为fill_fragment和 mma_sync 都在矩阵乘法文档中使用，所以我们只需要编写3个 intrinsic。

Scheduling the Computation 调度计算

要在TVM中使用张量核，我们必须将计算调度到特定的结构中，以匹配张量的内禀（intrinsic）。和传统的GPU程序一样，我们也可以使用共享内存来提高速度。如果你有任何关于blocking和共享内存的问题，请参考如何在GPU上优化卷积。
在这个例子中，每个块包含2x4的warp，每个warp调用4x2的TensorCore指令。因此，每个warp的输出形状是64x32，每个块输出128x128标题。由于共享内存空间的限制，我们一次只加载2个blocks (2x128x128 tiles)。
Note:

Warp-level Operation
注意，所有的TensorCore指令都是warp级指令，这意味着warp中的所有32个线程应该同时执行这条指令。使theadIdx.x extent=32是解决这个问题的最简单方法之一。然后我们可以绑定threadidx.x指向任何循环，除了那些直接或间接包含TensorCore intrinsic的循环。还要注意，它不是唯一的解决方案。我们要做的唯一一件事就是确保所有的线程都能在同一时间调用TensorCore。
