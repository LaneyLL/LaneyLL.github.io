MySQL基本

0. 关系型数据库和非关系型数据库的区别？

  • 关系型数据库的优点
    • 容易理解，因为它采取了关系模型来组织数据；
    • 可以保持数据的一致性；
    • 数据更新的开销比较少；
    • 支持复杂查询（带where子句的查询）。
  • 非关系型数据库的优点
    • 不需要经过SQL层的解析，读写效率高；
    • 基于键值对，数据的扩展性很好；
    • 支持多种数据类型的存储，如图片、文档等。

1. 介绍一下数据库三范式？

  • 第一范式：数据库表中所有的字段值都是不可分解的原子值。

    假设数据库系统中需要用到“地址”这个属性，本来直接将“地址”属性设计成一个数据库表的字段就行。但是如果系统经常“地址”属性中的“城市部分，那么就非要将”地址“这个属性重新拆分成省份、城市、详细地址等多个部分进行存储。

  • 第二范式：第二范式基于第一范式，且要求：数据库表中的每一列都和主键相关，且不能只和主键的某一部分相关（主要针对联合主键而言），即不存在部分依赖；

    下表是以:订单编号&商品编号作为联合主键。这样在该表中的商品名称、单位、商品价格等信息不与该表的订单编号相关，而仅仅是与商品编号相关。

  • 第三范式：基于第二范式，数据表中的每一列都和主键直接相关，即不存在传递依赖；

    比如在设计一个订单数据表的时候，可以将客户编号作为外键和订单表建立相应的关系。而不可以在表中添加关于客户其他信息（比如姓名、所属公司等）的字段。

2. MySQL数据引擎有哪些？

MySQL查看所有的数据引擎：

MySQL常用引擎包括：MYISAM、Innodb、Memory、MERGE,

  • MYISAM
    以select、insert为主的应用基本上可以使用这引擎。
    • 优点：全表锁，拥有较高的执行速度，占用空间少；
    • 缺点：不支持事务、不支持外键、并发性能差。
  • Innodb
    Innodb引擎提供了对数据库ACID事务的支持，并且实现了SQL标准的四种隔离级别，它的设计目标是处理大容量数据库系统。
    • 优点：行级锁，支持事务，支持自动增长列，支持外键约束，并发性能强
    • 缺点：占用空间是MYISAM的2倍，处理效率相对也更低
  • Memory
    主要用于内容变化不频繁的代码表。
    • 优点：全表锁，存储在内存中，默认使用Hash，检索效率非常高
    • 缺点：会占用和数据量成正比的内存空间，mysql重启时会失效，不适合精确查找
  • MERGE
    是一组MYISAM表的组合。

2.2 为什么InnoDB使用自增id作为主键？

  • 如果不使用自增主键，如身份证号、学号，每次插入主键的值近似于随机，因此每次新纪录都要被插到现有索引的中间，某个位置（涉及到B+tree分裂等），频繁地移动、分页操作造成了大量的碎片；
  • 如果使用自增主键，那么每次插入新的记录，记录就会顺序添加到当前索引节点的后续位置，当一页写满，就会自动开辟一个新的页。

3 说说InnoDB和MyISAM的区别？

3.1 说说InnoDB与MyISAM在B+树索引方式区别？

  • MyISAM，B+tree叶节点的data域存放的是数据记录的地址，在索引检索时，如果指定的Key存在，则取出data域的值，然后以data域的值为地址读取相应的数据记录，这被称为非聚簇索引；

  • InnoDB，树的节点data域保存了完整的数据记录，这个索引的key是数据表的主键（自增id),而其他索引都叫做辅助索引，辅助索引的data域存储相应记录主键的值而不是记录地址。

    • 在根据主索引搜索时，直接找到Key所在的节点即可取出数据；
    • 在根据辅助索引查找时，则需要取出主键的值，再走一遍的主索引。
