程序员应如何理解标准库

常用的printf函数以及C++中cout函数到底是在哪里实现的？

ISO组织对外发布的标准中包含两个部分内容：

  1. C/C++支持哪些特性
  2. C/C++API，程序员可以直接在他们的C/C++程序中直接调用这些API，这些API就被称为标准库（Standard Library）
    标准中只定义了API，并不包括实现，那么是谁实现彼岸准考中定义的API呢？

C/C++标准库的实现

API除了一些数学计算功能，还有像文件读写、内存分配、线程创建等等相关的API的实现，必须借助相应操作系统提供的功能，这个过程就是借助系统调用（System Call），我们所写的代码很多是需要依赖操作系统的，操作系统化其实提供了很多功能，程序员使用这些功能的方式其实就是借助系统调用。
因此我们知道每个平台（操作系统）都有自特定的标准库实现，因为不同的操作系统提供的功能是不同的，提供的系统调用也是不同的。
现在我们就可以回答最开始提出的问题，printf函数以及C++中cout函数的实现是在标准库中。

标准库在哪里？怎样使用？

libc.so.6就是标准库，标准库的实现被打包到了动态链接库中。编译器gcc在编译程序是默认情况下就自动链接了标准库。

Linux的标准库实现

Linux下标准库的实现被称为GNU C Library，也被称为glibc，在Linux平台中使用最为广泛。
关于COO的标准库实现在libstdc++，在linux平台中使用Idd工具就能看到这个标准库。

Windows标准库实现

Windows标准库实现是和微软的官方编译器Visual Studio绑定在一起的，该标准库曾被称为C/C++ Run-time Library（CRT）
从Windows95开始，微软以MSVCRT+版本号.DLL的命名来实行发布，到了1997年，将其简化为了MSVCRT.DLL。
从Visual Studio 2015之后，Windows中C/C++标准库被称为了Universal C Runtime Library (Universal CRT，简称UCRT)，即UCRTBASE.DLL，此后Windows标准库开始同Win10一起发布。
